## Goal
Build and publish an n8n workflow that receives TradingView POST alerts at
`/praxis-signal`, validates the JSON payload, writes valid signals to
`/Volumes/Sensidine/Praxis.build/signals/<ts>-<signal_id>.json`, and returns
appropriate HTTP responses (200 / 400 / 500).

## Nodes used (with discriminators)
1. `n8n-nodes-base.webhook` v2.1 — "TradingView Signal Webhook" — POST, path
   `praxis-signal`, `responseMode: responseNode`.
2. `n8n-nodes-base.code` v2 (mode `runOnceForAllItems`, JS) — "Validate Signal
   Payload" — checks required fields, side enum, qty>0, numeric price, ISO ts;
   builds sanitized filename (`:` replaced with `-`) and emits payload as binary.
3. `n8n-nodes-base.if` v2.3 — "Is Payload Valid" — boolean true on `$json.valid`.
4. `n8n-nodes-base.readWriteFile` v1.1 (operation `write`) — "Write Signal File"
   — `onError: continueErrorOutput` so write failures fall through to 500.
5. `n8n-nodes-base.respondToWebhook` v1.5 — "Respond 200 OK" — JSON
   `{ok:true, file:<path>}`.
6. `n8n-nodes-base.respondToWebhook` v1.5 — "Respond 400 Bad Request" — JSON
   `{ok:false, error:$json.error}`.
7. `n8n-nodes-base.respondToWebhook` v1.5 — "Respond 500 Write Error" — JSON
   `{ok:false, error:<message>}`.

## Validation output (final)
```
{ "valid": true, "nodeCount": 7,
  "warnings": [
    { "code": "INVALID_EXPRESSION_PATH",
      "message": "'Respond 500 Write Error' parameter 'responseBody' uses $json.error but no predecessor outputs this field." }
  ] }
```
Warning is a static-analysis false-positive: at runtime the readWriteFile node's
error output (via `onError: continueErrorOutput`) injects `$json.error` with
shape `{ message, ... }`. Verified against n8n SDK error-output convention.
